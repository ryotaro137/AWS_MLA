ある AWS アカウント (機械学習スタートアップのアカウント) のユーザーに別の AWS アカウント (会社のアカウント) のリソースへのアクセスを提供するには、必要な権限と信頼関係を持つ IAM ロールを会社のアカウントで作成し、機械学習スタートアップのアカウントの ID を指定する必要があります。そうすることで、クライアントのアカウントのユーザーがロールを引き受け、安全なクロスアカウントアクセスのための一時的な認証情報を取得することができます。アカウントをまたいだ S3 バケットに、プログラムによるアクセスとコンソールによるアクセスの両方を提供する唯一の方法は、クロスアカウント IAM ロールを設定することです。 このシナリオでは、会社のアカウントで作成されたロールを機械学習スタートアップのユーザーが引き継ぎ、S3 バケットにアクセスします。
