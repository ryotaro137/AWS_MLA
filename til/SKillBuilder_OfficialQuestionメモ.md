AWS Certified Machine Learning Engineer - Associate (MLA-C01) の試験範囲である **4つのドメイン** に基づいて気合いで分類しました！弱点補強や試験直前の振り返りにガンガン活用してください💪🔥

## 🎯 第1分野: データ準備とセキュリティ (Domain 1: Data Preparation for Machine Learning)
**データの収集、変換、ETL、機密データの処理に関する分野です。**

* **AWS Glue を用いた ETL と機械学習フレームワーク**
  * AWS Glue はデータ準備・変換用のフルマネージド ETL サービス。
  * TensorFlow や PyTorch などの機械学習フレームワーク向け Docker イメージはデフォルトで提供されていないため、手動でカスタムスクリプトにインポートして処理を行う必要があり、運用オーバーヘッドが発生する点に注意！
* **Amazon Comprehend による PII 検出・削除**
  * テキストドキュメント（英語・スペイン語等）のユーザーとのやりとりから、**個人情報 (PII) を検出・削除・匿名化**する用途に最適。データプロテクションの強力な味方！

## 🧠 第2分野: 機械学習モデルの開発 (Domain 2: ML Model Development)
**アルゴリズムの選定、トレーニング手法、チューニング、デバッグに関する領域です。**

* **SageMaker の入力モード (Fast File Mode等) 🚀速度勝負！**
  * **ファイルモード**: データをコンテナのローカルにDLして開始。
  * **パイプモード**: データをストリーミングで直接連携、パフォーマンスが向上！
  * **高速ファイルモード (Fast File Mode)**: 両方のいいとこ取り！S3データソースを識別するだけでDLはせず、全体の読み込み完了前にトレーニングを開始可能 (起動時間が劇的に短縮！)。運用オーバーヘッド最小で導入可能！
* **SageMaker with TensorBoard**
  * モデルトレーニング中の「中間テンソル（アクティベーションや勾配）」を視覚化・分析できる！
  * 適合率の低下要因などのデバッグや、モデルのさらなる最適化に大活躍🔥
* **SageMaker AMT (自動モデルチューニング)**
  * **ウォームスタート**: 以前のジョブを出発点にできる（`TRANSFER_LEARNING`設定により入力データやハイパーパラメータ範囲の変更も可能）。
  * **早期停止 (Early Stopping)**: 「自動」に設定すれば、目標メトリクス（正解率など）の移動平均中央値と比較し、見込みがないトレーニングを自動で停止してくれる🔥
* **RAG構成とベクトルデータベース (OpenSearch × Titan)**
  * **Amazon Titan Embeddings G1 - Text** 等のモデルで意味のあるベクトル表現を作成。
  * **OpenSearch Service** をベクトルデータベースとして使用し、n次元空間での最近傍探索（ベクトル検索）を効率的に実現。RAGの基盤として超重要！

## 🚀 第3分野: ML モデルのデプロイとオーケストレーション (Domain 3: ML Model Deployment and Orchestration)
**推論インフラストラクチャ、パイプライン、CI/CD、スケーリングに関する分野です。**

* **SageMaker Pipeline と EventBridge (バッチ推論の完全自動化！)**
  * モデルの呼び出しが定期的（例: 週に一度等）な場合は、常時起動のリアルタイム推論ではなく、**バッチ変換** が最もコスト効率が高い！
  * EventBridge を使用し、スケジュールルールで Pipeline の実行を自動化（カスタムイベントパターンの作成は不要）。
* **SageMaker サーバーレスエンドポイントの制限事項に注意⚠️**
  * リアルタイムのリクエスト応答に対応しているが、**VPCの設定ができない**。
  * 処理時間は **最大60秒** のため、推論タイムアウトリスクがあるユースケースではNG！
* **Amazon ECS (Fargate) を使用した推論の課題**
  * サーバーレスホスティングは可能だが、モデルを含めた独自のコンテナイメージ構築と最適化が必要。
  * SageMakerエンドポイントに比べて、管理と独自のオーバーヘッドが追加で重くのしかかってしまう。
* **SageMaker SDK の "独自コンテナを使用する" オプション**
  * R言語などで記述された既存の機械学習モデルを SageMaker 上に取り込む場合、既存モデルの **Dockerfile を用意するだけ** で対応可能。運用オーバーヘッドを極限まで最小化できる強み！
* **SageMaker エンドポイントの Auto Scaling (需要スパイクに対抗🔥)**
  * イベントによるスパイクに対応するため、**「スケジュールされたスケーリング」**（イベント前の計画的な容量増加）と **「ターゲット追跡スケーリングポリシー」**（メトリクスに応じた動的スケーリング）を組み合わせて万全の体制を構築する！

## 🛡️ 第4分野: ML ソリューションのオペレーション (Domain 4: ML Solution Operations)
**監視、運用、セキュリティ、アクセス制御、持続的なモデルパフォーマンス維持に関する分野です。**

* **SageMaker ネットワーク分離で鉄壁のセキュリティ🛡️**
  * デフォルトではコンテナのインターネットアクセスが有効なので、「ネットワーク分離」を有効にして、インターネットおよび外部アクセスを完全にブロックする。
* **S3 クロスアカウントアクセスの権限と IAM ロール**
  * あるアカウント（会社）のリソース（S3など）を別のアカウント（スタートアップ）に提供するには、会社側で **必要な権限を持つ IAM ロール** を作成し、信頼関係を設定する。
  * スタートアップユーザーがそのロールを引き受ける (AssumeRole) ことで、長期的な認証情報(アクセスキー等)を使用せず、一時的認証情報で安全にコンソール・プログラムを利用できる！
* **SageMaker Model Monitor (説明可能性モニタリング)**
  * `ModelExplainabilityMonitor` クラスを使用して「特徴量の貢献度のベースライン」を生成する。
  * 特徴量貢献度の **ドリフト** 発生の有無を評価し、CloudWatch と連携していつでも通知を受け取れるように運用する！
