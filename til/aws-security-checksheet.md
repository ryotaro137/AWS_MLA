# 実務で使えるAWSセキュリティチェックシート

[2021年版]AWSセキュリティ対策全部盛り[初級から上級まで] (*DevelopersIO*) より抽出・作成

## 1. 初級編：AWSをセキュアに始める・システムを作る

### アカウントの初期設定・監視設定（最低限やること）
- [ ] ルートユーザーの使用を避け、MFA（多要素認証）を設定しているか？
- [ ] 日常の作業用にIAMユーザーを作成し、MFAを設定しているか？
- [ ] 以下の必須セキュリティサービスを有効化しているか？
  - [ ] **AWS CloudTrail**（API操作の記録）
  - [ ] **AWS Config**（構成変更の記録）
  - [ ] **Amazon GuardDuty**（脅威検知）
  - [ ] **AWS Security Hub**（セキュリティ基準のチェック）
  - [ ] **IAM Access Analyzer**（意図しないリソース共有の特定）
  - [ ] **Amazon Detective**（根本原因の調査分析）
- [ ] AWS Well-Architected Framework のセキュリティの柱を理解しているか？

### システム構築のセキュリティ（構成のベストプラクティス）
- [ ] 責任共有モデルを理解し、S3などのマネージドサービスを積極的に活用しているか？
- [ ] パブリックサブネットにEC2を配置していないか？
- [ ] EC2へのSSH接続を最小限にし、代わりに**AWS Systems Manager (SSM) Session Manager**を利用しているか？
- [ ] アウトバウンド通信はNAT GatewayまたはVPC Endpointを経由させているか？
- [ ] 単一障害点をなくすため、マルチAZやAuto Scaling構成を採用しているか？
- [ ] ACM（AWS Certificate Manager）を利用して通信をHTTPS（SSL/TLS暗号化）にしているか？
- [ ] CloudFront + S3を利用し、静的コンテンツ配信やDDoS保護を行っているか？
- [ ] [AWS WAF] を適用し、アプリケーションレイヤーの攻撃から保護しているか？
- [ ] EC2内のOSやミドルウェアのパッチは常に最新に保たれているか（脆弱性管理）？
- [ ] EC2のセキュリティグループで必要最低限のポートのみを開放しているか？
- [ ] 各種ログ（OS、アプリなど）はCloudWatch LogsやS3に外部出力しているか？
- [ ] マルウェア対策（例：Cloud One Workload Security等の活用）を行っているか？
- [ ] EC2やLambda等に付与するIAM Roleは「最小権限」にしているか（SSRF対策）？
- [ ] 必要に応じて、メタデータへのアクセスにIMDSv2を利用しているか？
- [ ] コンテナ基盤を利用する場合、自前で管理せず**ECS / EKS**を利用しているか？

## 2. 中級編：AWSのセキュリティを維持する・インシデントに対応する

### セキュリティの維持（継続的なチェック）
- [ ] **AWS Security Hub**における「AWS Foundational Security Best Practices」等の基準を有効にし、スコアを定期的に確認しているか？
- [ ] **AWS Config (Config Rules)** を利用して、非準拠のリソース変更を検知しているか？
- [ ] 検知されたセキュリティ上の問題に対して、自動修復（SSM Automationとの連携など）の仕組みを検討・導入しているか？
- [ ] IAM Access Analyzer や Permissions Boundary を利用して、権限が肥大化していないか監視しているか？

### インシデント対応（検知と復旧）
- [ ] インシデント発生時の連絡体制や対応マニュアル（NIST CSF等の対応・復旧計画）は整備されているか？
- [ ] **Amazon GuardDuty** を活用して以下の脅威を監視しているか？
  - [ ] EC2での不正なコインマイニングやマルウェア感染（C&Cサーバー通信）
  - [ ] IAMの不正利用（不正ログイン、CloudTrail無効化）
  - [ ] S3への不審なアクセス
- [ ] インシデント調査時に **Amazon Detective** を利用して効率的に追跡・分析できる準備があるか？

## 3. 上級編：組織全体でガバナンスを効かせる

### マルチアカウントガバナンスと自動化
- [ ] **AWS Organizations** を利用してマルチアカウントを集約管理しているか？
- [ ] Service Control Policy (SCP) を活用し、組織全体で以下の制限を行っているか？
  - [ ] 特定リージョンの使用禁止
  - [ ] CloudTrail の無効化禁止
  - [ ] GuardDuty の無効化禁止
  - [ ] S3バケットのパブリック公開禁止
  - [ ] 必須タグ付けの強制
- [ ] **AWS SSO** (現: AWS IAM Identity Center) を利用して、マルチアカウントへのIDとアクセス権を一元管理しているか？
- [ ] **AWS CloudFormation StackSets** を用いて、マルチアカウント/マルチリージョンにセキュリティの仕組みを自動展開しているか？
- [ ] AWS Systems Manager Explorer などを利用して、マルチアカウントの運用情報をダッシュボードに集約しているか？
- [ ] AWS Control Tower の導入などにより、組織レベルでの請求一元管理やコンプライアンス制御を達成できているか？
