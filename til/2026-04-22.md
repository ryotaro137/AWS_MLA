## 学習内容

### 機械学習におけるバイアス（Fairness Measures for Machine Learning in Finance）
論文「Fairness Measures for Machine Learning in Finance」にて提唱されている、モデルにバイアスが発生する6つの代表的な要因：

1. **Biased labels（バイアスの影響を受けたラベル）**
   人間の持つバイアスに起因し、データセットに蓄積されるもの。複数のラベラーが関与する公開データセット（警察のデータや世論調査などのデータ）において特に発生しやすい。
2. **Biased features / Curation bias（偏った特徴量 / キュレーションバイアス）**
   特定の特徴量を選択したり、逆に除外したりする過程で直接的または間接的に発生するバイアス。特定のグループが不利になるような特徴量を選択し、有利になるものを除外してしまうことで発生（意図的であるか無意識であるかを問わない）。
3. **Objective function bias（目的関数のバイアス）**
   損失関数がデータセット内の特定タイプの外れ値（アウトライアー）に過剰に焦点を当ててしまった場合などに生じるバイアス。モデラーが意図せずバイアスを注入してしまう原因になる。
4. **Homogenization bias（同質化バイアス）**
   機械が後続のモデルを訓練するためのデータを生成し、バイアスを永続化・増幅させてしまう現象。フィードバックループを通じて将来のモデルや意思決定、データそのものを一層偏らせる。
5. **Active bias（アクティブバイアス）**
   事実ではない作り話（例：フェイクニュース）から生じるバイアスで、情報に触発された人々の推論や意見、意思決定に偏りをもたらす。悪意のある発信源や確証バイアスなどを見極めて管理する必要がある。
6. **Unanticipated machine decisions（予期せぬ機械の意思決定）**
   機械学習が「文脈（コンテキスト）」を欠いたまま最適解を導き出すことによって生じるバイアス。モデルの目的や制約にコンテキストが注入されていないため、文脈を無視した妥当性のない極端な答え（例：赤字最小化のために常軌を逸した関税を導き出し貿易戦争を引き起こすなど）を生成してしまう。
